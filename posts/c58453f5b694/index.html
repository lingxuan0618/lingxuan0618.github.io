賦值運算子

基於其右方的運算元的值賦予其左方的運算元。最簡單的賦值運算子是等於(=)

名稱 簡化的運算子 意義
賦值 x = y x = y
加法賦值 x += y x = x + y
減法賦值 x -= y x = x - y
乘法賦值 *= y x = x * y
除法賦值 x /= y x = x / y
餘數賦值 x %= y x = x % y
指數賦值 x **= y x = x ** y
左移賦值 x <<= y x = x << y
右移賦值 x >>= y x = x >> y
無號右移賦值 x >>>= y x = x >>> y
位元 AND 賦值 x &= y x = x & y
位元 XOR 賦值 x ^= y x = x ^ y
位元 OR 賦值 x |= y x = x | y

比較運算子

比較運算子會比較運算元，並基於比較的結果回傳邏輯值(true or false)

運算子 描述 會回傳 True 的例子
等於(==) 假如運算元相等就回傳 True 1 == ‘1’
不等於(!=) 假如運算元不相等就回傳 True 1 != 2
嚴格等於(===) 假如運算元具有相同型態且相等則回傳 True 1 === 1
嚴格不等於(!==) 假如運算元具有相同型態但不相等，或是具有不同型態，回傳 True。 1 !== 2
大於(>) 假如左方運算元大於右方運算元，回傳 True 2 > 1
大於或等於(>=) 假如左方運算元大於或等於右方運算元，回傳 True 2 >= 1
小於(<) 假如左方運算元小於右方運算元，回傳 True 1 < 2
小於或等於(<=) 假如左方運算元小於或等於右方運算元，回傳 True 1 <= 2

通常在判斷的時候，都會使用嚴格等於或者嚴格不等於，
一般的相同運算子，在兩邊的的型別不一樣時，是會強制轉型成一樣的

但嚴格等於跟嚴格不等於，必須要兩邊的型別相同才會被判斷成true，所以避免開發時，因為沒注意到型別，而導致判斷出問題，建議都使用嚴格等於或者嚴格不等於來進行比較及判斷

算術運算子

運算子 描述 範例
加法(+) 回傳兩個運算元相加的結果 2 + 5 = 7
減法(-) 回傳兩個運算元相減的結果 7 - 5 = 2
乘法(*) 回傳兩個運算元相乘的結果 7 * 5 = 35
除法(/) 回傳兩個運算元相除的結果 35 / 7 = 5
取餘數(%) 回傳兩個運算元相除後的餘數 7 % 3 回傳 1 (7 / 3 = 2…1 餘數是1)

邏輯運算子

JS中的&& 和 || 在判斷兩邊數值時，會先判斷左邊的數值，如果已經成立，則不會繼續判斷右邊，所以不會占用任何效能

使用布林型態做判斷，會回傳布林；但如果非布林型態，則會回非布林型態的值(運算式1或2)

運算子 用法 描述
邏輯 AND (&&) 運算式1 && 運算式2 假如運算式1可以被轉換成false的話，回傳運算式1; 否則，回傳運算式2。因此，&&只有在兩個運算元都是True時才會回傳True，否則回傳false。
邏輯 OR (||) 運算式1 || 運算式2 假如運算式1可以被轉換成true的話，回傳運算式1; 否則，回傳運算式2。因此，||在兩個運算元有任一個是True時就會回傳True，否則回傳false。
邏輯 NOT (!) !運算式 假如單一個運算元能被轉換成 True 時，回傳false ， 不然回傳 true。
